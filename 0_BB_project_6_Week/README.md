# 📈 Bollinger Bands Weekly Trading Bot (Sandbox)

Автоматизированная рекомендательная система (Proof of Concept) для торговли акциями на Московской бирже через API Тинькофф Инвестиций. Система использует стратегию пробоя **Полос Боллинджера** на **недельном таймфрейме**.

> ⚠️ **Внимание:** Проект работает в режиме **ПЕСОЧНИЦЫ (Sandbox)**. Все сделки выполняются на виртуальном счете без использования реальных денег. Код предоставляется "как есть" и не является финансовой рекомендацией.

## 🚀 Особенности

- **Недельный таймфрейм (W1):** Анализ данных раз в неделю, что снижает уровень рыночного шума и количество ложных сигналов.
- **Стратегия Bollinger Bands:**
  - 🟡 **ВНИМАНИЕ:** Цена закрытия ниже нижней полосы после падения под SMA(20).
  - 🟢 **КУПИ:** Подтверждение входа (цена ниже цены сигнала "ВНИМАНИЕ", но выше SMA).
  - 🔵 **ДОКУПИ:** Усреднение позиции при дальнейшем падении цены.
  - 🔴 **ПРОДАЙ:** Выход из позиции при возврате цены выше SMA(20).
- **Двойные уведомления:** Отправка сигналов в Telegram (через прокси/обход блокировок) и на Email (SMTP Mail.ru).
- **История и аналитика:** Сохранение всех сделок в PostgreSQL и Excel, построение графика эквити.
- **Модульная архитектура:** Разделение загрузки данных, анализа и исполнения сделок.

🛠️ Установка и настройка
1. Требования
Python 3.9+
PostgreSQL
Аккаунт в Тинькофф Инвестициях (доступ к API)
Аккаунт Mail.ru (для отправки email)
Telegram Bot (опционально)

2. Установка зависимостей
pip install -r requirements.txt

3. Настройка переменных окружения (.env)
Создайте файл .env в корне проекта и заполните его данными:
# Tinkoff Invest API (Sandbox Token)
TINKOFF_TOKEN=t.YOUR_SANDBOX_TOKEN_HERE

# Database Configuration
DB_NAME=pg4
DB_USER=postgres
DB_PASSWORD=your_db_password
DB_HOST=localhost
DB_PORT=5432

# Email Notifications (Mail.ru)
EMAIL_LOGIN=bbweek@mail.ru
EMAIL_PASSWORD=YOUR_APP_PASSWORD_HERE
EMAIL_RECEIVER=bbweek@mail.ru

# Telegram (Optional)
TELEGRAM_BOT_TOKEN=YOUR_BOT_TOKEN
TELEGRAM_CHAT_ID=YOUR_CHAT_ID
💡 Важно: Для Mail.ru используйте "Пароль приложения", а не основной пароль от почты. Получить его можно в настройках безопасности почтового ящика.

4. Подготовка базы данных
Убедитесь, что PostgreSQL запущен. Таблицы создаются автоматически при первом запуске data_loader.py.
🚀 Запуск
Запуск всей системы осуществляется через главный скрипт:
python main.py
Что происходит при запуске:
Data Loader: Скачивает актуальные недельные свечи по списку тикеров и обновляет БД.
Signals Processor: Анализирует данные, находит паттерны и записывает сигналы в БД.
Email Notifier: Формирует и отправляет красивый HTML-отчет со списком активных сигналов на почту.
Trader Executor: Проверяет сигналы и исполняет сделки в песочнице Тинькофф.

📊 Торговая логика (Недельный таймфрейм)
Сигнал | Условие | Действие
ВНИМАНИЕ | Close < Lower Band AND предшествующее пересечение SMA сверху вниз | Уведомление, подготовка к покупке
КУПИ | Close < Close(ВНИМАНИЕ) AND Close < SMA(20) | Покупка на всю сумму операции (макс. 10 акций)
ДОКУПИ | Close < Средняя цена покупки (Harmonic Avg) | Докупка для усреднения позиции
ПРОДАЙ | Close > SMA(20) | Полная продажа позиции
Период SMA: 20 недель (~5 месяцев).
Полосы Боллинджера: 2 стандартных отклонения.
Риск-менеджмент: Максимум 5000 руб. на операцию, комиссия 0.3%.

⚠️ Известные ограничения
Блокировка Telegram: В РФ доступ к Telegram может быть ограничен. Рекомендуется использовать прокси или полагаться на Email-уведомления.
Режим песочницы: Сделки не влияют на реальный капитал. Для перехода на реальный счет необходимо изменить SANDBOX_MODE = False в config.py (требуется доработка риск-менеджмента).
Таймфрейм: Стратегия оптимизирована под долгосрочное удержание позиций (недели/месяцы). Не подходит для дейтрейдинга.
